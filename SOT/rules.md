SOT/rules.md는 프로젝트의 '헌법'인 CLAUDE.md보다 훨씬 **구체적이고 실무적인 '행동 지침'**을 담는 곳입니다. CLAUDE.md가 "무엇을 지켜야 한다"는 원칙이라면, rules.md는 **"이럴 땐 이렇게 행동해라"**라는 매뉴얼에 가깝습니다.

## 에러 대응 및 기록
- 동일한 에러가 2번 이상 반복되면 `SOT/logs.md`에 '재발 방지 대책'을 적고, `CLAUDE.md`의 규칙 수정을 건의한다.
  
- 금융 API 연동 코드를 짤 때는 반드시 재시도(Retry) 로직과 타임아웃 처리를 포함하라

## Python 코드 작성 규칙
- `@abstractmethod`를 사용하는 클래스는 반드시 `ABC`를 상속하라. ABC 없이 사용하면 추상 클래스로 인식되지 않아 인스턴스 생성이 차단되지 않는다.
