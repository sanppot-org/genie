# 프로젝트 실행 스킬 (Skills.md)

## 1. 신규 기능 구현 스킬 (Feature Implementation)
이 스킬은 새로운 기능을 추가할 때 호출한다.

### [단계 1: 맥락 파악]
- `omc`를 사용하여 관련 컴포넌트와 데이터 모델의 구조를 파악한다.
- `SOT/architecture.md`를 읽고 기존 설계 원칙을 확인한다.

### [단계 2: 구현 계획 수립]
- 코드를 수정하기 전, 마크다운 형식으로 '구현 계획서'를 작성하여 보고한다.
- 계획서에는 수정할 파일 목록, 데이터 흐름의 변화, 신규 추가되는 라이브러리를 포함한다.
- **[Lean 원칙]:** 코드를 최소한으로 유지할 방법을 계획에 반드시 포함한다.

### [단계 3: 코드 작성 및 검증]
- `CLAUDE.md`의 스타일 가이드에 따라 코드를 작성한다.
- 작성 후 스스로 코드를 리뷰하여 중복된 로직이 없는지 체크한다.

---

## 2. 코드 다이어트 스킬 (Code Cleanup & Refactor)
코드가 비대해지거나 중복이 발견될 때 호출한다.

### [단계 1: 탐색]
- 미사용 변수, 중복된 유틸리티 함수, 너무 비대해진 파일(150줄 이상)을 찾는다.
- `omc`를 활용해 프로젝트 전체에서 재사용 가능한 로직을 식별한다.

### [단계 2: 삭제 및 통합]
- 기능 변경 없이 코드 양을 줄이는 방향으로 리팩토링한다.
- 불필요한 테스트 코드는 과감히 통합하거나 제거한다.

---

## 3. 작업 종료 및 기록 스킬 (Post-Process)
모든 작업이 완료된 후 반드시 수행한다.

- 수정된 사항 중 기술적으로 중요한 결정은 `SOT/decisions.md`에 업데이트한다.
- 발생했던 에러와 해결법은 `SOT/logs.md`에 추가한다.
- **[SOT 동기화]:** 만약 `CLAUDE.md`의 규칙을 수정해야 할 필요가 보인다면 사용자에게 제안한다.

---
## 4. TDD 워크플로우

- 계획: 어떤 기능을 테스트할지 나열한다.
- AI 자가 검토: "이 테스트들이 정말 필요한가? 중복은 없는가?"를 스스로 묻고 불필요한 항목은 리스트에서 지운다.
- 실행: 남은 핵심 항목에 대해서만 TDD를 수행한다.

## 🔄 [Skill] 시스템 자가 진화 (Self-Evolution)

### 개요
`SOT/logs.md`에 기록된 시행착오를 분석하여, 동일한 실수가 반복되지 않도록 프로젝트의 실행 규칙(`CLAUDE.md`, `rules.md`, `AGENTS/*.md`)을 최적화하고 지능을 승격시킨다.

### 실행 조건 (Trigger)
- 하루의 작업을 마무리할 때
- 대규모 리팩토링이나 복잡한 버그 해결 직후
- 사용자가 "시스템 회고를 수행하라"고 요청할 때

### 작업 절차 (Steps)
1. **패턴 식별 (Log Analysis):** - `SOT/logs.md`의 최신 항목 10개와 `SOT/refactor_log.md`를 읽는다.
    - 반복되는 에러, 환경 설정 오류(버전 충돌 등), 혹은 비효율적인 구현 패턴을 식별한다.
2. **근본 원인 파악 (Context Mapping):**
    - 식별된 문제가 `SOT/context.md`에 명시된 비즈니스 목표나 기술 스택과 어떻게 충돌하는지 분석한다.
    - 규칙이 없어서 발생한 문제인지, 기존 규칙이 모호해서 발생한 문제인지 판단한다.
3. **업데이트 제안서 작성 (Proposal):**
    - **현상:** 반복되는 문제 요약
    - **원인:** 규칙의 부재 또는 모호함
    - **해결안:** `CLAUDE.md`, `rules.md`, 또는 특정 `AGENTS/*.md`에 추가/수정할 구체적인 규칙 문구
4. **사용자 승인 및 반영:**
    - 제안서를 사용자에게 보고하고 승인을 받는다.
    - 승인된 규칙을 해당 파일에 반영하고, 업데이트 사실을 `SOT/decisions.md`에 기록한다.

### 기대 효과
- AI가 점진적으로 사용자 맞춤형 '전문 비서'로 진화함.
- 동일한 에러로 인한 토큰 낭비 및 개발 시간 지연을 원천 차단함.
